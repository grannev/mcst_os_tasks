# Шаблон структуры репозитория с решениями задач

Автоматизированная система тестирования накладывает некоторые ограничения на оформление решения:

* Исходные тексты решения задачи N нужно расположить в директории taskN.
* Для кажодой задачи добавляется taskN/Makefile со следующим обязательным функционалом:
    - При вызове make, находясь в директории taskN, в ней собирается исполняемый файл taskN - решение задачи.
    - Для задач, написанных на C и C++, Makefile должен поддерживать переменные среды CFLAGS и CXXFLAGFS, добавляя флаги, указанные в этих переменных к опциям gcc и g++ соответственно.

В остальном предоставляется свобода, например, в Makefile можно добавить цели для очистки и тестировани, можно использовать отличные от make системы сборки "под капотом" Makefile. Реализованный функционал стоит отразить в README.
